0.3: Project Creation & Folder Setup

Good organization is key to a healthy project. First, create the Flutter project, then we'll set up our folders.

1. Run this command in your terminal:

flutter create habit_tracker

2. Open the `habit_tracker` folder. Inside `lib`, create a `controllers` folder. Our structure is now:

lib/
├── controllers/    # For our GetX controllers
├── models/         # For our data structures
├── screens/        # For each full screen of our app
├── widgets/        # For small, reusable UI components
└── main.dart

Chapter 01: The Main Screen - A Static View

In this chapter, we will build the visual shell of the main screen. It won't have any logic yet, but it will look correct. This "static-first" approach is a great way to build UIs.

1.1: The Habit Model

First, define what a "habit" is in our code. This is our data model.

lib/models/habit.dart
class Habit {
  final String name;
  final String description;
  bool isCompleted;

  Habit({
    required this.name,
    required this.description,
    this.isCompleted = false,
  });
}

1.2: The HabitListItem Widget

Next, we'll create a dedicated, reusable widget to display a single habit. It will need a function to call when the checkbox is tapped.

lib/widgets/habit_list_item.dart
import 'package:flutter/material.dart';

class HabitListItem extends StatelessWidget {
  final String habitName;
  final bool isCompleted;
  final Function(bool?)? onToggle;

  const HabitListItem({
    super.key,
    required this.habitName,
    required this.isCompleted,
    required this.onToggle,
  });

  @override
  Widget build(BuildContext context) {
    return ListTile(
      title: Text(habitName),
      leading: Checkbox(
        value: isCompleted,
        onChanged: onToggle,
      ),
    );
  }
}

1.3: Assembling the Main Screen

Now we create the main screen itself. For now, it will be a simple `StatelessWidget`.

lib/screens/main_screen.dart
import 'package:flutter/material.dart';

class MainScreen extends StatelessWidget {

  @override

        title: const Text('My Habits'),

      body: Center(child: Text("Habits will be loaded here!")),

1.4: Update main.dart to Show Our Screen

Finally, tell our app to show our new `MainScreen` on launch. We'll use `GetMaterialApp` instead of `MaterialApp` to enable GetX features.

lib/main.dart
import 'package:flutter/material.dart';
import 'package:get/get.dart';
import 'screens/main_screen.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return GetMaterialApp(
      title: 'Habit Tracker',
      theme: ThemeData.dark(),
      home: const MainScreen(),
    );
  }
}

Chapter 02: Reactive State with GetX

This is where the magic happens! We'll introduce GetX to manage our app's state. This means our UI will automatically update when our data changes, without needing `StatefulWidget` or `setState()`.

2.1: Add the GetX Package

Open your `pubspec.yaml` file and add `get` under dependencies. Then, run `flutter pub get` in your terminal.

pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  get: ^4.6.6 # Add this line
  # ... other dependencies

2.2: Create the HabitController

The Controller is the brain of our screen. It holds the data (the list of habits) and the logic to manipulate that data. Notice the .obs - this makes our list "observable", so GetX can watch it for changes.

lib/controllers/habit_controller.dart
import 'package:get/get.dart';
import '../models/habit.dart';

class HabitController extends GetxController {
  // .obs makes this list reactive.
  var habits = [].obs;

  @override
  void onInit() {
    super.onInit();
    // Load some dummy data when the controller is initialized
    habits.addAll([
      Habit(name: 'Read for 15 minutes', description: 'Expand the mind'),
      Habit(name: 'Go for a walk', description: 'Good for the body', isCompleted: true),
      Habit(name: 'Drink 8 glasses of water', description: 'Stay hydrated'),
    ]);
  }

  void toggleHabit(Habit habit) {
    habit.isCompleted = !habit.isCompleted;
    habits.refresh(); // Tell GetX that an item inside the list has changed.
  }
}

2.3: Refactor the Main Screen

We'll now connect our `MainScreen` to the `HabitController`. Get.put() creates and registers our controller. The Obx widget rebuilds its child whenever the `habits` list inside the controller changes.

lib/screens/main_screen.dart
import 'package:flutter/material.dart';
import 'package:get/get.dart';
import '../controllers/habit_controller.dart';
import '../widgets/habit_list_item.dart';

class MainScreen extends StatelessWidget {
  const MainScreen({super.key});

  @override
  Widget build(BuildContext context) {
    // Initialize our controller. GetX will manage its lifecycle.
    final HabitController controller = Get.put(HabitController());

    return Scaffold(
      appBar: AppBar(
        title: const Text('My Habits (GetX)'),
      ),
      // Obx listens to reactive variables in the controller and rebuilds on change.
      body: Obx(() => ListView.builder(
        itemCount: controller.habits.length,
        itemBuilder: (context, index) {
          final habit = controller.habits[index];
          return HabitListItem(
            habitName: habit.name,
            isCompleted: habit.isCompleted,
            onToggle: (value) {
              // We will implement this in the next step!
            },
          );
        },
      )),
      floatingActionButton: FloatingActionButton(
        onPressed: () {},
        child: const Icon(Icons.add),
      ),
    );
  }
}

2.4: Implement the Toggle Logic

The final step is to make the checkbox work. We simply call the `toggleHabit` method on our controller. GetX handles the rest!

lib/screens/main_screen.dart (Updated `itemBuilder`)
// Inside the ListView.builder's itemBuilder...
itemBuilder: (context, index) {
  final habit = controller.habits[index];
  return HabitListItem(
    habitName: habit.name,
    isCompleted: habit.isCompleted,
    onToggle: (value) {
      // Simply call the method on the controller.
      controller.toggleHabit(habit);
    },
  );
},

Milestone Reached!

Run the app again. The UI now loads its data from the controller. Best of all, when you tap a checkbox, the UI updates instantly and automatically. This is the power of reactive state management!

Chapter 00: Project Kickstart & Vision

0.3: Project Creation & Folder Setup

1. Run this command in your terminal:

flutter create habit_tracker

2. Open the `habit_tracker` folder. Inside lib, create a `controllers` folder. Our structure is now:

lib/
├── controllers/
├── models/
├── screens/
├── widgets/
└── main.dart

Why do we create this folder structure?

Organizing code makes it easier to find things and work efficiently. This structure separates our code based on its purpose, which is a core principle of good software design.

Chapter 01: The Main Screen - A Static View

1.1: Add the GetX Package

Open your pubspec.yaml file and add GetX under dependencies.

pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  get: ^4.6.6

1.2: The Habit Model

This data model will be a class with final properties.

lib/models/habit.dart
class Habit {
  final String name;
  final String description;
  bool isCompleted;

  Habit({
    required this.name,
    required this.description,
    this.isCompleted = false,
  });
}

1.3: The HabitListItem Widget

We'll create a reusable StatelessWidget that uses a ListTile and a Checkbox.

lib/widgets/habit_list_item.dart
import 'package:flutter/material.dart';

class HabitListItem extends StatelessWidget {
  final String habitName;
  final bool isCompleted;
  final Function(bool?)? onToggle;

  const HabitListItem({
    super.key,
    required this.habitName,
    required this.isCompleted,
    required this.onToggle,
  });

  @override
  Widget build(BuildContext context) {
    return ListTile(
      title: Text(habitName),
      leading: Checkbox(
        value: isCompleted,
        onChanged: onToggle,
      ),
    );
  }
}

1.4: Assembling the Main Screen

Now create the main screen using a Scaffold, AppBar, and ListView.

lib/screens/main_screen.dart
import 'package:flutter/material.dart';
import '../widgets/habit_list_item.dart';

class MainScreen extends StatelessWidget {
  const MainScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('My Habits'),
      ),
      body: ListView(
        children: [
          HabitListItem(habitName: 'Read for 15 minutes', isCompleted: true, onToggle: (val){}),
          HabitListItem(habitName: 'Go for a walk', isCompleted: false, onToggle: (val){}),
        ],
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {},
        child: const Icon(Icons.add),
      ),
    );
  }
}

1.5: Update main.dart

Tell our app to show `MainScreen` on launch. We'll use GetMaterialApp and define a ThemeData.

lib/main.dart
import 'package:flutter/material.dart';
import 'package:get/get.dart';
import 'screens/main_screen.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return GetMaterialApp(
      title: 'Habit Tracker',
      theme: ThemeData.dark(),
      home: const MainScreen(),
    );
  }
}

Chapter 02: Reactive State with GetX

We'll introduce GetX to manage our app's state, replacing `StatefulWidget`.

2.1: Create the HabitController

The GetxController separates logic from UI. The .obs makes our list "reactive."

lib/controllers/habit_controller.dart
import 'package:get/get.dart';
import '../models/habit.dart';

class HabitController extends GetxController {
  var habits = [].obs;

  @override
  void onInit() {
    super.onInit();
    habits.addAll([
      Habit(name: 'Read for 15 minutes', description: 'Expand the mind'),
      Habit(name: 'Go for a walk', description: 'Good for the body', isCompleted: true),
    ]);
  }

  void toggleHabit(Habit habit) {
    habit.isCompleted = !habit.isCompleted;
    habits.refresh();
  }
}

2.2: Refactor the Main Screen

We'll now connect our `MainScreen` to the `HabitController`. Get.put() creates our controller. The Obx widget rebuilds its child whenever the `habits` list changes.

lib/screens/main_screen.dart
import 'package:flutter/material.dart';
import 'package:get/get.dart';
import '../controllers/habit_controller.dart';
import '../widgets/habit_list_item.dart';

class MainScreen extends StatelessWidget {
  const MainScreen({super.key});

  @override
  Widget build(BuildContext context) {
    final HabitController controller = Get.put(HabitController());

    return Scaffold(
      appBar: AppBar(title: const Text('My Habits (GetX)')),
      body: Obx(() => ListView.builder(
        itemCount: controller.habits.length,
        itemBuilder: (context, index) {
          final habit = controller.habits[index];
          return HabitListItem(
            habitName: habit.name,
            isCompleted: habit.isCompleted,
            onToggle: (value) => controller.toggleHabit(habit),
          );
        },
      )),
      floatingActionButton: FloatingActionButton(
        onPressed: () {},
        child: const Icon(Icons.add),
      ),
    );
  }
}

Milestone Reached!

Run the app. The UI now loads its data from the controller and updates instantly when you tap a checkbox!

GetX is a powerful, lightweight framework for Flutter that handles state management, dependency injection, and route management with minimal boilerplate code.

The pubspec.yaml file is the manifest for your Flutter project. It defines metadata like the project name and version, and most importantly, it lists all the external packages (dependencies) your project needs, like GetX.

The lib (short for library) folder is where all your Dart code for the application lives. The execution of your app starts from the `main.dart` file inside this folder.

A class is a blueprint for creating objects. Our `Habit` class defines that every habit object *must* have a name, description, and completion status. This enforces a consistent structure for our data.

The final keyword declares a variable that can only be assigned a value once. It must be initialized when it's declared or in the class constructor. This is useful for properties that should not change after the object is created, promoting immutability and safer code.

A Widget is the fundamental building block of all Flutter UIs. Everything you see on screen—a button, text, a list, even the padding around them—is a widget. You build complex screens by composing simple widgets together, like stacking LEGO bricks.

A StatelessWidget is a widget that cannot change its own state once built. It's perfect for UI elements that are purely presentational and depend only on the configuration data passed into them from their parent.

A ListTile is a convenient, pre-styled Material Design widget that creates a fixed-height row. It's commonly used in lists and typically contains text as well as leading or trailing icons, like a checkbox or an arrow.

A Checkbox is a simple widget that allows the user to make a binary choice. It displays a small box that can be either checked or unchecked, representing a true/false value.

A Scaffold widget provides a standard visual layout structure for a Material Design screen. It gives you APIs for showing app bars, drawers, floating action buttons, and a `body` for the main screen content. It's the foundation for almost every screen in a Flutter app.

An AppBar is the toolbar that appears at the top of a screen, typically within a `Scaffold`. It usually contains the screen title, action buttons, and sometimes tabs.

A ListView is a core widget that arranges its children sequentially in a scrollable list. The standard `ListView` constructor is best for a small, fixed number of children because it builds all of them at once.

GetMaterialApp is a replacement for Flutter's standard `MaterialApp`. It's the root widget of your app when using GetX. It provides all the same functionality but also sets up the routing and dependency injection systems that GetX needs to work.

ThemeData is a class that holds the color and typography values for a theme. By defining a `ThemeData` at the top of your app (in `GetMaterialApp`), you can ensure all child widgets inherit the same visual styling, like colors and fonts, for a consistent look and feel.

A GetxController is a class from the GetX package designed to hold your business logic and state variables, separating them from your UI code. It has a lifecycle (like `onInit()` and `onClose()`) and is automatically managed by GetX's dependency injection system.

Get.put() is GetX's dependency injection method. It creates an instance of a controller (like `HabitController`) and registers it with GetX, making it available to all child widgets in the widget tree. GetX intelligently manages when to keep the controller in memory and when to dispose of it.

The ListView.builder is a highly performant way to build lists. Unlike a standard `ListView`, it only creates the list items that are currently visible on screen. As the user scrolls, it recycles widgets, which is essential for long or infinite lists as it saves a tremendous amount of memory.

The .obs is a GetX extension that turns any variable into a "reactive" or "observable" stream. When an `.obs` variable changes, any reactive widget (like `Obx`) that uses it will automatically rebuild, ensuring the UI is always in sync with the state.

Obx is a GetX reactive widget that takes a function returning a widget. It automatically subscribes to any `.obs` variables used within that function. Whenever one of those variables changes, `Obx` efficiently rebuilds only itself, not the whole screen.

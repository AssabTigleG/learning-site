0.3: Project Creation & Folder Setup

Good organization is key to a healthy project. First, create the Flutter project, then we'll set up our folders.

1. Run this command in your terminal to create the project:

flutter create habit_tracker

2. Open the `habit_tracker` folder in your code editor. Inside the `lib` folder, create the following subfolders. This keeps our code organized by feature.

lib/
├── models/         # For our data structures (like the Habit model)
├── screens/        # For each full screen of our app
├── widgets/        # For small, reusable UI components
└── main.dart

Chapter 01: The Main Screen - A Static View

In this chapter, we will build the complete visual shell of the main screen. It won't do anything yet, but it will look correct. This "static-first" approach is a great way to build UIs.

1.1: The Habit Model

Before we can display a habit, we need to define what a "habit" is in our code. This is our data model.

lib/models/habit.dart
class Habit {
  final String name;
  final String description;
  bool isCompleted;

  Habit({
    required this.name,
    required this.description,
    this.isCompleted = false,
  });
}

1.2: The HabitListItem Widget

Next, we'll create a dedicated, reusable widget to display a single habit in the list. This makes our code much cleaner.

lib/widgets/habit_list_item.dart
import 'package:flutter/material.dart';

class HabitListItem extends StatelessWidget {
  final String habitName;
  final bool isCompleted;

  const HabitListItem({
    super.key,
    required this.habitName,
    required this.isCompleted,
  });

  @override
  Widget build(BuildContext context) {
    return ListTile(
      title: Text(habitName),
      leading: Checkbox(
        value: isCompleted,
        onChanged: (value) {
          // Logic will be added in a later chapter!
        },
      ),
    );
  }
}

1.3: Assembling the Main Screen

Now we create the main screen itself, using our new `HabitListItem` widget in a `ListView`.

lib/screens/main_screen.dart
import 'package:flutter/material.dart';
import '../widgets/habit_list_item.dart';

class MainScreen extends StatelessWidget {
  const MainScreen({super.key});

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(
        title: const Text('My Habits'),
        backgroundColor: Colors.grey[900],
      ),
      body: ListView(
        children: const [
          HabitListItem(habitName: 'Read for 15 minutes', isCompleted: true),
          HabitListItem(habitName: 'Go for a walk', isCompleted: false),
          HabitListItem(habitName: 'Drink 8 glasses of water', isCompleted: false),
        ],
      ),
      floatingActionButton: FloatingActionButton(
        onPressed: () {
          // Navigation will be added later!
        },
        child: const Icon(Icons.add),
      ),
    );
  }
}

1.4: Update main.dart to Show Our Screen

Finally, we need to tell our app to show our new `MainScreen` when it launches, instead of the default Flutter counter app.

lib/main.dart
import 'package:flutter/material.dart';
import 'screens/main_screen.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Habit Tracker',
      theme: ThemeData.dark().copyWith(
        primaryColor: Colors.blue,
        scaffoldBackgroundColor: Colors.black,
      ),
      home: const MainScreen(),
    );
  }
}
